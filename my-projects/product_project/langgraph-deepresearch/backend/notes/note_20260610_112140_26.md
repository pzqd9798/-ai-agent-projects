# 任务 3: 项目作品集设计

任务 3 已完成分析。核心发现：
1. **项目选型：从“堆砌技术”到“解决商业问题”**：企业级面试官更看重业务闭环思维。一个简单的“智能客服”或“商品推荐助手”，如果包含完整的“意图识别->知识检索->回复/下单”链路和价值（如成本优化、效率提升），远比一个技术堆砌但缺乏业务逻辑的复杂项目更吸引人。
2. **技术栈设计：分层渐进，突出“工程落地”能力**：多数本科生项目停留在调用API和简单Demo。脱颖而出的关键在于展示工程实践能力：如RAG的向量库选型（Milvus vs. Elasticsearch）、记忆压缩策略（滑动窗口/摘要压缩）、Agent死循环检测、异步任务（MQ）和成本控制（Token统计、缓存）。
3. **面试话术：用“STAR法则”讲透“踩坑与填坑”**：面试官关注的是你在项目中遇到的最大挑战及解决方案。例如，不要只说“用了RAG”，而要描述“如何解决RAG知识滞后问题，通过增量索引将文档更新延迟从1小时降到5分钟”。这是普通本科生和名校生拉开差距的关键。
4. **差异化利器：打造一份“可交互的能力看板”**：用Notion等工具制作个人能力看板，集成GitHub代码、项目文档（README）、在线Demo、技术笔记。这比静态PDF简历的视觉冲击力强得多，在AI Agent筛选简历中也更占优势（链接和关键词密度高）。
5. **项目来源：巧用“逆向工程”和“开源重构”**：不必从零构思。可以基于GitHub上高质量的Agent开源项目进行二次开发和业务场景适配，但必须在文档中清晰说明自己的改进点和价值，这同样被视为具备“微创新”和“学习能力”的优秀品质。
