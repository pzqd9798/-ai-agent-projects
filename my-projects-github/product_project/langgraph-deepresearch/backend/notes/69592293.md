## 任务 3: 异步库与生态 - 新增调研总结

### 关键发现
1. **核心原理**：以 `asyncio` 单线程事件循环 + `async/await` 实现协作式多任务，高效应对 I/O 密集型场景；相比多线程，上下文切换开销极低。
2. **Web 框架选型**：FastAPI 基于 ASGI、类型注解，原生支持异步视图、混合同步/异步函数，自动生成 API 文档，性能接近 Node.js，是当前构建异步 REST/WebSocket 的首选。
3. **数据库异步驱动**：须使用 `asyncpg` (PostgreSQL)、`asyncmy` (MySQL) 等专用驱动，或通过 `databases`、SQLAlchemy `asyncio` 扩展实现异步 ORM；需在启动/关闭时管理引擎和连接池生命周期。
4. **HTTP 客户端**：`httpx.AsyncClient` 和 `aiohttp` 提供全异步 HTTP 调用，支持连接池和并发聚合请求；可配合 `asyncio.gather` 并行调用外部服务，显著降低延迟。
5. **高级生态与工程实践**：异步任务队列（如 Celery + 事件循环桥接、ARQ）、AnyIO 结构化并发、Asyncer 封装等进一步扩展异步能力；同步阻塞代码可用 `asyncio.to_thread` 桥接，适用于遗留系统渐进式改造。
