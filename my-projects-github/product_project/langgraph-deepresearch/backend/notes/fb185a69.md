## 任务总结

### 关键发现
1. **Semaphore 安全模式**：始终优先使用 `async with semaphore:`，替代手动 `acquire/release`；用 `BoundedSemaphore` 防止过度释放。结合 `asyncio.wait_for` 实现带超时的获取，但取消任务时需谨慎处理，以免泄露信号量。
2. **Queue 是更安全的并发边界**：`asyncio.Queue`（`maxsize=n`）配合固定数量的 worker 实现生产者-消费者模式，不仅能限制并发数，还能防止无限制的任务产生和内存暴涨，并天然支持背压。`PriorityQueue` 可处理紧急任务。
3. **异步错误处理解耦**：将失败消息推入专用 `asyncio.Queue`，由独立协程消费并执行退避重试，主流程不阻塞，提升整体吞吐与弹性。
4. **取消安全的结构化并发**：可取消任务中直接使用 Semaphore 存在根本性风险，推荐采用 Queue + 固定 worker 模式，并通过 `CancelledError` 捕获、队列 join 和 worker 取消的严格顺序实现优雅关闭。
5. **调试与监控**：锁争用和并发瓶颈不易复现，应通过埋点监测排队延迟、信号量等待时间，及早发现性能问题。
